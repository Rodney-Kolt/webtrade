# Fundamental OTC Signal Engine

> **WARNING: These signals are generated by a model. Always confirm manually.
> Never connect this engine directly to live trading.**

A 24/7 Python signal engine that monitors real macroeconomic data and news
sentiment to generate manual trading alerts for Pocket Option OTC instruments.
Alerts are delivered as plain-text messages to your Telegram Saved Messages.

**The engine never connects to Pocket Option or any broker API.**
Every alert ends with: *"MANUAL TRADE ONLY – Open Pocket Option and place this
trade yourself."*

---

## How it works

```
Every 15 minutes:
┌─────────────────────────────────────────────────────┐
│  1. Finnworlds API  →  high-impact macro events      │
│     actual vs consensus  →  currency bias map        │
│                                                      │
│  2. NewsAPI  →  top business headlines               │
│     keyword scoring  →  RISK_ON / RISK_OFF / NEUTRAL │
│                                                      │
│  3. Combine both  →  directional bias per currency   │
│     Map to OTC pairs  →  CALL or PUT signals         │
│                                                      │
│  4. Cooldown check  →  skip duplicates (1 hour)      │
│                                                      │
│  5. Send plain-text alert to Telegram Saved Messages │
└─────────────────────────────────────────────────────┘
```

---

## Signal logic

### Macro rules (Finnworlds)
| Event result | Currency bias |
|---|---|
| Actual > Consensus (employment, GDP, PMI, rate decision…) | BULLISH |
| Actual < Consensus | BEARISH |
| Conflicting events same currency | NEUTRAL |

### Sentiment rules (NewsAPI)
| Keywords detected | Sentiment | Effect |
|---|---|---|
| recession, crash, sell-off, crisis… | RISK_OFF | USD/JPY/CHF BULLISH, AUD/NZD/CAD/EUR/GBP BEARISH |
| growth, rally, optimism, recovery… | RISK_ON | Risk currencies BULLISH, safe havens BEARISH |

### OTC pair mapping
When a currency has a clear directional bias, all OTC pairs containing that
currency receive a CALL or PUT signal accordingly.

## Setup

### 1. Get API keys

| Service | URL | Free tier |
|---|---|---|
| Finnworlds | https://finnworlds.com | Macroeconomic calendar |
| NewsAPI | https://newsapi.org | 100 requests/day |
| Telegram | https://my.telegram.org/apps | API ID + hash |

### 2. Configure environment

```bash
cp .env.example .env
# Edit .env with your real keys
```

### 3. Install dependencies

```bash
pip install -r requirements.txt
```

### 4. First-time Telegram login

Run once locally to complete phone verification:

```bash
python app.py
```

Telegram will send a code to your phone. Enter it when prompted.
A `signal_engine.session` file is created — keep it safe.

### 5. Run

```bash
python app.py
```

The engine starts, Flask listens on port 10000, and the scheduler runs every
15 minutes.

## Deploying on Render

### 1. Push to GitHub (without secrets)

Ensure `.gitignore` excludes `.env` and `*.session`.

```bash
git add app.py requirements.txt .env.example README.md .gitignore
git commit -m "Initial commit: OTC Signal Engine"
git push
```

### 2. Create a Render Web Service

| Setting | Value |
|---|---|
| Build command | `pip install -r requirements.txt` |
| Start command | `python app.py` |
| Port | `10000` |

### 3. Add environment variables

In Render dashboard → Environment → add all variables from `.env.example`.

### 4. Upload session file

In Render dashboard → Environment → Secret Files:
- Path: `signal_engine.session`
- Content: paste the binary content of your local `signal_engine.session`

### 5. Keep alive with UptimeRobot

1. Go to https://uptimerobot.com (free)
2. Add a new HTTP(S) monitor
3. URL: `https://your-app.onrender.com/health`
4. Interval: every 5 minutes

This prevents Render's free tier from spinning down the service.

## Environment variables reference

| Variable | Required | Default | Description |
|---|---|---|---|
| `TG_API_ID` | ✅ | — | Telegram API ID |
| `TG_API_HASH` | ✅ | — | Telegram API hash |
| `TG_PHONE` | ✅ | — | Your phone number |
| `TG_MY_CHAT_ID` | ✅ | — | Destination (`me` or numeric ID) |
| `FINNWORLDS_API_KEY` | ✅ | — | Finnworlds API key |
| `NEWS_API_KEY` | ✅ | — | NewsAPI key |
| `SCAN_INTERVAL_SECONDS` | ❌ | `900` | Seconds between scans |
| `COOLDOWN_SECONDS` | ❌ | `3600` | Duplicate suppression window |
