# OTC Signal Aggregator Bot

> **WARNING: These signals are generated by a model. Always confirm manually.
> Never connect this engine directly to live trading.**

A private Telegram bot that aggregates **fundamental, technical, news, and
social** signals for Pocket Option OTC assets and delivers validated alerts
directly to you. You trade manually — the bot never touches any broker API.

---

## Architecture

```
┌─────────────────────────────────────────────────────────────┐
│                     Background Threads                       │
│                                                             │
│  WebSocket Thread          Scheduler Thread (every 15 min) │
│  └─ PO price stream        ├─ fetch_fundamentals()          │
│     └─ build 1-min candles ├─ fetch_news()                  │
│        └─ RSI + EMA        ├─ fetch_social()                │
│                            ├─ run_technical_analysis()      │
│                            └─ combine_signals()             │
│                                  └─ push to subscribers     │
├─────────────────────────────────────────────────────────────┤
│  Telegram Bot Thread       Flask Thread (port 10000)        │
│  /start /latest            /health  ← UptimeRobot pings    │
│  /subscribe /data /status  /status  ← JSON status dump     │
└─────────────────────────────────────────────────────────────┘
```

## Signal combination rules

A signal fires **only when at least 2 sources agree** on direction:

| Example combination | Result |
|---|---|
| RSI oversold (technical) + RISK_ON news | CALL |
| USD bullish macro + high USD social chatter | CALL on USD pairs |
| Fundamental bearish EUR + RISK_OFF news | PUT on EUR pairs |
| Only 1 source agrees | No signal |

Confidence levels:
- **HIGH** – 3 or more sources agree
- **MEDIUM** – exactly 2 sources agree

## Setup

### 1. Create a Telegram bot

1. Open Telegram, search for **@BotFather**
2. Send `/newbot`, follow the prompts
3. Copy the **bot token**

### 2. Get API keys

| Service | URL |
|---|---|
| Finnworlds | https://finnworlds.com |
| NewsAPI | https://newsapi.org |

### 3. Get Pocket Option SSID (optional, for technical signals)

1. Log into Pocket Option in your browser
2. Open DevTools → Application → Cookies
3. Find the cookie named `ssid` and copy its value
4. Paste into `POCKET_SSID` env var

Without `POCKET_SSID`, the bot still works using fundamentals + news + social.

### 4. Configure environment

```bash
cp .env.example .env
# Edit .env with your real values
```

### 5. Install and run

```bash
pip install -r requirements.txt
python app.py
```

## Deploy on Render

### 1. Push to GitHub (no secrets)

```bash
git add app.py requirements.txt .env.example README.md .gitignore
git commit -m "Initial commit: OTC Signal Aggregator"
git push
```

### 2. Create Render Web Service

| Setting | Value |
|---|---|
| Root directory | `aggregator` |
| Build command | `pip install -r requirements.txt` |
| Start command | `python app.py` |
| Port | `10000` |

### 3. Add environment variables

In Render dashboard → Environment → add all variables from `.env.example`.

### 4. Keep alive with UptimeRobot

1. Go to https://uptimerobot.com (free)
2. Add HTTP(S) monitor → `https://your-app.onrender.com/health`
3. Interval: every 5 minutes

## Environment variables

| Variable | Required | Default | Description |
|---|---|---|---|
| `TELEGRAM_BOT_TOKEN` | ✅ | — | Token from @BotFather |
| `FINNWORLDS_KEY` | ✅ | — | Finnworlds API key |
| `NEWS_API_KEY` | ✅ | — | NewsAPI key |
| `POCKET_SSID` | ❌ | *(empty)* | PO session cookie for WebSocket |
| `SCAN_INTERVAL_SECONDS` | ❌ | `900` | Seconds between scans |
| `SIGNAL_COOLDOWN_SECONDS` | ❌ | `60` | Duplicate suppression window |
